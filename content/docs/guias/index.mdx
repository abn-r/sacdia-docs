Guias practicas para los distintos aspectos del ciclo de desarrollo de SACDIA.

## Guias disponibles

| Guia | Descripcion |
|------|-------------|
| [Deployment](/docs/guias/deployment) | Como desplegar el backend (NestJS), el panel admin (Next.js) y la app mobile (Flutter) a produccion |
| [Developer Workflow](/docs/guias/developer-workflow) | Flujo recomendado para trabajar con documentacion y desarrollo, con checklists pre y post implementacion |
| [Admin Integration](/docs/guias/admin-integration) | Como integrar el frontend admin con los contratos de backend vigentes |
| [Idea to Spec](/docs/guias/idea-to-spec) | Como convertir una idea en una especificacion ejecutable con requirements, design y tasks |

## Como usar estas guias

Las guias son documentos operativos: son cortos, directos y orientados a la accion. No explican el "por que" del sistema — para eso existen los documentos de arquitectura y steering.

Antes de implementar algo nuevo, revisa:

1. [Developer Workflow](/docs/guias/developer-workflow) para entender el flujo de trabajo
2. [Idea to Spec](/docs/guias/idea-to-spec) si necesitas definir el alcance
3. [Admin Integration](/docs/guias/admin-integration) si vas a conectar el frontend con el backend
4. [Deployment](/docs/guias/deployment) cuando estes listo para publicar
