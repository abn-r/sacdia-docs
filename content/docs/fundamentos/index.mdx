Los fundamentos son el punto de partida obligatorio para entender cualquier parte de SACDIA. Antes de explorar endpoints, pantallas o módulos técnicos, es necesario comprender el lenguaje del sistema y las decisiones que lo gobiernan.

## Documentos de esta sección

### Modelo del dominio

Define el lenguaje oficial, los conceptos centrales y las reglas de interpretación del dominio. Ningún endpoint, tabla, permiso, pantalla o documento técnico puede redefinir los conceptos establecidos aquí.

[Ver Modelo del dominio](/docs/fundamentos/dominio)

### Identidad del sistema

El propósito de SACDIA, el problema que resuelve, para quién existe y qué lo diferencia de una solución administrativa genérica.

[Ver Identidad del sistema](/docs/fundamentos/identidad)

### Arquitectura del sistema

Cómo SACDIA materializa su dominio en componentes técnicos con responsabilidades diferenciadas: dominio, backend, admin web, app móvil y datos.

[Ver Arquitectura del sistema](/docs/fundamentos/arquitectura)

### Runtime SACDIA

Descripción del estado técnico actual del sistema: stack documentado, topología, superficie API, capacidades, autorización y persistencia.

[Ver Runtime SACDIA](/docs/fundamentos/runtime)

### Decisiones clave

Las decisiones estructurales que afectan de forma duradera la interpretación, organización o evolución de SACDIA. Preservan memoria útil y evitan rediscusiones sin contexto.

[Ver Decisiones clave](/docs/fundamentos/decisiones-clave)

### Source of Truth

La jerarquía documental vigente del proyecto. Define qué fuentes mandan, cuáles solo apoyan y cuáles no pueden usarse como estado actual.

[Ver Source of Truth](/docs/fundamentos/source-of-truth)
