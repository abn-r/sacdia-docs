> Cada nueva pantalla, componente o modal debe seguir estas definiciones.

## Que es este sistema de diseno

El sistema de diseño de SACDIA Admin define las reglas visuales, los componentes reutilizables y los patrones de interaccion que garantizan una experiencia coherente en todo el panel administrativo.

No es un conjunto de sugerencias: es la fuente de verdad. Cualquier nueva pantalla, componente o flujo debe ajustarse a lo documentado aqui antes de entregarse.

## Cuando usarlo

Consulta este sistema siempre que vayas a:

- Crear una pantalla nueva
- Agregar un componente nuevo
- Revisar cómo debe verse un estado de carga, error o vacio
- Decidir entre variantes de un componente
- Implementar dark mode en algun elemento
- Definir espaciado, tipografia o colores

## Regla absoluta

Toda interfaz nueva o existente usa componentes de `@/components/ui/*` (shadcn/Radix) para elementos interactivos visibles.

Si se necesita un componente que no existe, instalar via:

```bash
npx shadcn@latest add <componente>
```

**Excepciones tipicas permitidas**:
- `<input type="hidden">` para formularios o server actions
- Elementos nativos requeridos por APIs del navegador o integraciones third-party sin wrapper equivalente
- Casos puntuales documentados en PR con justificacion tecnica (accesibilidad o compatibilidad)
