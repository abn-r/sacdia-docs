SACDIA no usa RBAC puro. Usa un modelo de **RBAC contextual** que combina roles, permisos granulares, scope territorial y asignaciones de club para resolver qué puede hacer cada actor en cada contexto.

## Idea central

La autorización de SACDIA se entiende en dos planos:

- **Grants**: todo lo que el usuario tiene asignado potencialmente (roles globales, asignaciones de club).
- **Effective**: lo que el backend ya resolvió para la sesión actual (permisos vigentes, scope activo).

Los clientes siempre consumen `effective`. Los grants se usan para matrices administrativas y selectores de contexto.
