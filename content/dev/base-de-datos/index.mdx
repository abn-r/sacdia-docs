SACDIA utiliza PostgreSQL hosteado en Supabase como base de datos principal. El acceso a la base de datos desde el backend se realiza exclusivamente a traves de Prisma ORM.

## Autoridad del Schema

La fuente de verdad del modelo de datos es el archivo `schema.prisma` en el repositorio `sacdia-backend`. Esta documentacion es una referencia humana generada a partir de ese archivo y se sincroniza manualmente.

```
sacdia-backend/
└── prisma/
    ├── schema.prisma      ← FUENTE DE VERDAD
    └── migrations/        ← Historial de migraciones Prisma
```

## Convenciones Clave

- **ORM**: Siempre usar Prisma para queries — nunca raw SQL directo
- **Naming**: snake_case para tablas y columnas
- **PKs**: `{tabla}_id` UUID para tablas principales
- **Timestamps**: `created_at` y `modified_at` en todas las tablas
- **Soft delete**: Campo `active` BOOLEAN en lugar de DELETE fisico (en la mayoria de tablas)
- **Transacciones**: Usar `prisma.$transaction()` para operaciones relacionadas
