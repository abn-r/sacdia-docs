SACDIA es un monorepo de tres aplicaciones interconectadas: un backend REST, un panel de administración web y una app móvil. Esta sección cubre todo lo que un desarrollador necesita para contribuir al proyecto.

## Stack Tecnológico

| Componente | Tecnología | Versión |
|-----------|-----------|---------|
| Backend | NestJS + Prisma | 11.x / 7.x |
| Admin Panel | Next.js + shadcn/ui | 16.x |
| App Móvil | Flutter + Riverpod | 3.x |
| Base de Datos | PostgreSQL (Neon) | 15.x |
| Autenticación | Better Auth | self-hosted |
| Storage | Cloudflare R2 | — |
| Cache | Redis (Upstash) | — |
| Push Notifications | Firebase FCM | — |

## Arquitectura General

El backend expone una API REST versionada (`/api/v1/`) que consumen tanto el panel admin como la app móvil. La autenticación es JWT via Better Auth. El almacenamiento de archivos usa Cloudflare R2.

```
sacdia-admin (Next.js)
       |
       |  HTTP + JWT
       v
sacdia-backend (NestJS)  <-->  PostgreSQL (Neon)
       ^                  <-->  Redis (cache)
       |                  <-->  Cloudflare R2 (storage)
  HTTP + JWT              <-->  Firebase FCM (push)
       |
sacdia-app (Flutter)
```

## Convenciones de Commits

Todos los repositorios usan Conventional Commits:

```
feat:      Nueva funcionalidad
fix:       Corrección de bug
docs:      Documentación
refactor:  Refactoring sin cambio de comportamiento
test:      Tests
chore:     Tareas de mantenimiento
```

## Flujo de Trabajo Git

- `main` — producción, protegida
- `development` — integración activa
- `feature/*` — features individuales, se mergean a `development`
- `fix/*` — hotfixes
